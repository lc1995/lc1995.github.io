《Unity Shader 入门精要》Chapter 9 – Reading Note

Before, we have only one light source in our scene, and it’s always directional light. However, in practice, we will deal with more light sources, which are also more complex. Also, we would like to obtain the shadow.

Rendering Path

Rendering path defines how lighting is applied into Unity Shader. Or let’s say, Unity Shader will prepare many data about Lights and Shadows for you after you set the correct rendering path.

Unity supports different Rendering Paths. You should choose which one you use depending on your game content and target platform / hardware. Different rendering paths have different performance characteristics that mostly affect Lights and Shadows.

Rendering Path

Most of the cases, one project only uses one rendering path. This can be set in Edit -> Project Settings -> Player -> Other Settings -> Rendering Path.

Sometimes we will use multiple rendering path. We can set the rendering path for different camera, and this will overwrite the global setting. We can also set different rendering path for different Pass by LightMode.

The following is the supported tags of LightMode:

  • Always: Pass will always be rendered, but without calculating any lighting.
  • ForwardBase: Used in Forward Rendering, this Pass will calculate ambient light, the most important directional light, per-vertex/SH light and Lightmaps.
  • ForwardAdd: Used in Forward Rendering, this Pass will calculate other per-pixel light. Each Pass corresponds to a light source.
  • Deferred: Used in Deferred Shading. This Pass will render G-buffer.
  • ShadowCaster: Render the depth infomation of objects into a shadowmap or a depth map.
  • PrepassBase: Used in Legacy Deferred, this Pass will render normal and pow of specular.
  • PrepassFinal: Used in Legacy Deferred, this Pass will render the final color by combining texture, light and emissive.
  • Vertex, VectexLMRGBM and VertexLM: Used in Legacy Vertex Lit Rendering.

Forward Rendering in Unity

In Unity, we have 3 ways to deal with lighting: per-vertex, per-pixel and spherical harmonics. The Render Mode tells whether a light source is important or not. If it’s set to important, the light will be regarded as a per-pixel light.

In forward rendering, when we render one object, Unity will sort all lights according to their importances. Their importances depend on their setting(important/not important), and how much they will affect the object(e.g distance, light itensity). Then some of the light sources will be processed per pixel, then up to 4 sources will be processed per vertex, the remaining sources will be processed in SH.

The judgment rule can be shown as following:

  • Lights that have their Render Mode set to Not Important are always per-vertex or SH.
  • Brightest directional light is always per-pixel.
  • Lights that have their Render Mode set to Important are always per-pixel.
  • If the above results in less lights than current Pixel Light Count Quality Setting, then more lights are rendered per-pixel, in order of decreasing brightness.

In Unity, we have two different types of forward rendering: Base Pass and Additional Pass.

Usually the common tags and render settings can be shown as following:

  • #pragma multi_compile_fwdbase(/fwdadd) guarantees that Unity will build all Shader versions for different platforms. It will process different rendering logic and pass necessary built-in variables to Shader.
  • Base Pass supports some lit characteristics like lightmap.
  • By default, Base Pass supports shadow(only directional light), Additional Pass doesn’t support. We can use #pragma multi_compile_fwdadd_fullshadows manually in Addtional Pass to turn on shadow.
  • Ambient and emissive is calculated in Base Pass.
  • We will turn on Blend in Additional Pass. By default, we use Blend One One.
  • In forward rendering, a Unity Shader usually defines a Base Pass and an Additional Pass. Each Base Pass will run one time, and each Additional Pass will run multiple times dependent on how many lights are affecting the object.

Built-in Variables and Functions

Here we only show the most common variables and functions.

  • _LightColor0: float4 | The color of per-pixel light.
  • _WorldSpaceLightPos0: float4 | The position of per-pixel light. If it’s a directional light, the w component is 0. Otherwise it’s 1.
  • unity_WorldToLight: float4x4 | Transform matrix from world space to light space. This can be used to sample cookie and lit attenuation texture.
  • float3 UnityWorldSpaceLightDir(float4 v): Return the light direction in world space.
  • float3 UnityWorldSpaceViewDir(float4 v): Return the view direction in world space.

The others can be checked from Official Document.

Vertex Lit Rendering Path

Vertex lit rendering path is the one with highest performance and lowest requirement on hardware. However, it’s also the one with the worst effect. It doesn’t support those per-pixel effect like shadow, normal texture, specular with high precision(per-pixel) and so on. Now it’s legacy.
